Introduzione

Introduzione

Cos'è ASKY:

ASKY è una piattaforma di chat avanzata basata sull'intelligenza artificiale, progettata per offrire un'esperienza di comunicazione interattiva e personalizzata. La piattaforma combina un'interfaccia utente intuitiva con un backend robusto per garantire sicurezza, scalabilità e facilità d'uso. Le tecnologie utilizzate includono HTML, CSS, JavaScript (JS) per il frontend e PHP per il backend, con un collegamento a un database (DB) per la gestione dei dati degli utenti.

Da dove viene "ASKY"?

Il nome ASKY è un gioco di parole tra il verbo inglese "to ask", ovvero "chiedere", il codice ASCII, che nella lingua inglese viene pronunciato come "aschi", ed il nostro logo, un husky (ideato e stilizzato da Marcello Sergio Merolla - 5Abio).

Perché un'AI?

Le AI sono il futuro del lavoro, in ogni settore, ne abbiamo avuto la prova diretta durante l'esperienza di PCTO da ITConsulting.
L'AI è in grado di svolgere compiti ripetitivi e noiosi, liberando tempo per compiti più importanti e specifici.
Inoltre l'AI, essendo addestrata su grandi quantità di dati, è in grado di fornire risposte più precise e veloci anche per cose che
riguardano regole e protocolli di sicurezza sul lavoro.

Introduzione Tecnologie Usate

Tecnologie Utilizzate

Front End

  • HTML: Utilizzato per strutturare il contenuto delle pagine web;
  • CSS: Utilizzato per lo stile e il design delle pagine, garantendo un'interfaccia utente moderna e responsive;
  • JavaScript (JS): Utilizzato per aggiungere interattività dinamica alle pagine, come la gestione delle chat e la validazione dei form.

Back End

  • PHP: Utilizzato per la logica di backend, inclusa la gestione delle richieste, l'autenticazione degli utenti e l'interazione con il database;
  • Database (DB): Utilizzato per memorizzare le informazioni degli utenti, come credenziali di accesso, chat salvate e impostazioni.
    Il database è progettato per essere scalabile e sicuro.

HTML

HTML (HyperText Markup Language) è il linguaggio standard per la creazione di pagine web. Utilizza tag per definire la struttura e il contenuto delle
pagine web. Venne creato da Tim Berners-Lee nel 1991 ed è ancora ampiamente utilizzato per la creazione di siti web statici e dinamici. Ecco un esempio di codice HTML che definisce una semplice pagina web: 

<!DOCTYPE html>
<html>
    <head>
        <title>La mia web Page</title>
    </head>
    <body>
        <h1>Ciao, Mondo!</h1>
        <p>Questo è un paragrafo.</p>
    </body>
</html>

CSS

CSS (Cascading Style Sheets) è un linguaggio di stile utilizzato per definire l'aspetto e il layout delle pagine web. Consente di separare il contenuto
HTML dalla presentazione, consentendo una maggiore flessibilità e controllo sul design del sito. CSS è essenziale per creare un'interfaccia utente
moderna e responsive.
Ecco un esempio di codice CSS che definisce lo stile di un paragrafo HTML: 

p {
        color: red;
        text-align: center;
    }

JavaScript

JavaScript è un linguaggio di programmazione ad alto livello che aggiunge interattività e dinamicità alle pagine web. Viene utilizzato per creare
effetti visivi, animazioni, validazione dei form, interazioni utente e molto altro. JavaScript è essenziale per creare un'esperienza utente coinvolgente
e interattiva.
Ecco un esempio di codice JavaScript che mostra un messaggio di benvenuto all'utente: 

var name = prompt("Qual è il tuo nome?");
alert("Ciao, " + name + "! Benvenuto sul nostro sito!");

PHP

PHP (PHP: Hypertext Preprocessor) è un linguaggio di scripting lato server ampiamente utilizzato per lo sviluppo web. Viene utilizzato per creare
pagine web dinamiche, gestire le richieste degli utenti, interagire con i database e molto altro. PHP è un linguaggio potente e flessibile che
consente di creare applicazioni web complesse e scalabili. Questo va inserito all'internmo delle pagine HTML in questo modo: 

<?php
    $name = "Mondo";
    echo "Ciao, $name!";
?>

Database

Un database è un sistema di archiviazione dei dati che consente di memorizzare, organizzare e recuperare informazioni in modo efficiente. I database
vengono utilizzati per memorizzare le informazioni degli utenti, come le credenziali di accesso, le chat salvate ed altro ancora.
I database sono progettati per essere scalabili, sicuri e affidabili.

Personalità

La personalità di ASKY non è altro che un prompt (definibile promopt di default o promt 0) predefinito dato in pasto al modello dell'AI che, da esso, ricaverà tutte le informazioni su di se e su come interagire con l'utente oltre che al contesto in cui operare. La personalità è un aspetto molto importante per un'AI, in quanto permette di rendere l'interazione più umana e naturale. Inoltre, una personalità ben definita può aiutare l'utente a sentirsi più a suo agio e a creare un legame più forte con l'AI. La personalità di ASKY è basata sui concetti fondamentali della sicurezza sul lavoro con tanto di parole e frasi chiave che esso potrà usare mentre dialoga con l'utente.

LM STUDIO ACCESSI ED ENDPOINT ALL’AI

GET e POST sono metodi JS che ci permettono di dialogare direttamente con l'IA passandole e ottenendo da essa direttamente in chiaro i messaggi, in formato JSON.

Invio del messaggio all'AI (POST request)

async function sendMessage() {
if (isTyping) return;

const userMessage = elements.userInput.value.trim();
if (!userMessage) return;

// Resetta l'input
elements.userInput.value = '';
elements.userInput.style.height = 'auto';
elements.sendButton.disabled = true;

// Aggiunge il messaggio dell'utente alla chat
addMessage(userMessage, 'user-message');

isTyping = true;

// Mostra indicatore di digitazione
const typingIndicator = document.createElement('div');
typingIndicator.className = 'typing-indicator';
typingIndicator.innerHTML = '<span></span></span></span></span></span>';
elements.chatMessages.appendChild(typingIndicator);
elements.chatMessages.scrollTop = elements.chatMessages.scrollHeight;

try {
    // Crea l'array dei messaggi includendo la personalità se disponibile
    const messages = [];

    // Aggiungi la personalità come messaggio di sistema se disponibile
    if (personality) {
        messages.push({
            role: "system",
            content: personality
        });
    }

    // Aggiungi il messaggio dell'utente
    messages.push({
        role: "user",
        content: userMessage
    });

    // Invia la richiesta all'endpoint dell'AI
    const response = await fetch(settings.endpoint, {
        method: 'POST',
        headers: {
            'Content-Type': 'application/json'
        },
        body: JSON.stringify({
            model: settings.model === 'custom-model' ? settings.customModel : settings.model,
            messages: messages,
            temperature: settings.temperature,
            max_tokens: settings.maxTokens === -1 ? null : settings.maxTokens
        })
    });

    if (!response.ok) {
        throw new Error(`Errore HTTP: ${response.status}`);
    }

    const data = await response.json();
    const aiResponse = data.choices[0].message.content;

    // Rimuove l'indicatore di digitazione
    typingIndicator.remove();

    // Aggiunge la risposta dell'AI
    addMessage(aiResponse, 'ai-message');
} catch (error) {
    console.error('Errore durante l\'invio del messaggio:', error);
    typingIndicator.remove();

    // Mostra errore API
    elements.apiError.textContent = `Errore: ${error.message}`;
    elements.apiError.style.display = 'block';
    setTimeout(() => {
        elements.apiError.style.display = 'none';
    }, 5000);
} finally {
    isTyping = false;
    elements.sendButton.disabled = false;
    autoSaveChats(); // Salvataggio automatico dopo l'invio del messaggio
}
}

Questa parte del codice invia i messaggi dell'utente all'endpoint dell'AI e gestisce la risposta.

Salvataggio e caricamento delle chat (GET/POST requests)

// Funzione per salvare tutte le chat nel database
function saveAllChats() {
    if (chats.length === 0) {
        showNotification('Nessuna chat da salvare', true);
        return;
    }

    // Mostra un indicatore di caricamento
    showNotification('Salvataggio in corso...', false);

    // Prepara i dati da inviare
    const formData = new FormData();
    formData.append('action', 'saveChats');
    formData.append('chats', JSON.stringify(chats));

    // Invia la richiesta al server
    fetch('test.php', {
        method: 'POST',
        body: formData
    })
    .then(response => response.json())
    .then(data => {
        if (data.success) {
            showNotification(data.message || 'Tutte le chat salvate con successo');
        } else {
            showNotification(data.message || 'Errore durante il salvataggio', true);
        }
    })
    .catch(error => {
        console.error('Errore:', error);
        showNotification('Errore di connessione durante il salvataggio', true);
    });
}

// Funzione per caricare le chat dal database
function loadChatsFromDatabase() {
    // Mostra un indicatore di caricamento
    showNotification('Caricamento chat in corso...', false);

    // Invia la richiesta al server
    fetch('test.php?action=loadChats')
    .then(response => response.json())
    .then(data => {
        if (data.success) {
            if (data.chats && data.chats.length > 0) {
                chats = data.chats;
                currentChatIndex = 0;
                updateChatList();
                loadChatMessages(chats[currentChatIndex]);
                showNotification('Chat caricate con successo');
            } else {
                chats = [];
                currentChatIndex = -1;
                updateChatList();
                clearChatMessages();
                showNotification(data.message || 'Nessuna chat trovata');
            }
        } else {
            showNotification(data.message || 'Errore durante il caricamento', true);
        }
    })
    .catch(error => {
        console.error('Errore:', error);
        showNotification('Errore di connessione durante il caricamento', true);
    });
}

Queste funzioni gestiscono il salvataggio e il caricamento delle chat dal database utilizzando richieste HTTP.

Auto-salvataggio delle chat (POST request)

function autoSaveChats() {
if (chats.length === 0) {
    return;
}

// Cancella timeout precedente per evitare troppi salvataggi ravvicinati
if (saveTimeout) {
    clearTimeout(saveTimeout);
}

// Imposta un nuovo timeout di 1 secondo per il salvataggio
saveTimeout = setTimeout(() => {
    // Prepara i dati da inviare
    const formData = new FormData();
    formData.append('action', 'saveChats');
    formData.append('chats', JSON.stringify(chats));

    // Invia la richiesta al server
    fetch('test.php', {
        method: 'POST',
        body: formData
    })
    .then(response => response.json())
    .then(data => {
        if (!data.success) {
            console.error('Errore durante il salvataggio automatico:', data.message);
        }
    })
    .catch(error => {
        console.error('Errore di connessione durante il salvataggio automatico:', error);
    });
}, 1000); // Ritardo di 1 secondo per evitare troppe richieste
}

Questa funzione gestisce l'auto-salvataggio delle chat, inviando periodicamente le chat al server per il salvataggio.

Vantaggi e svantaggi dei vari modelli

gemma-2-2b-it-GGUF
    •   Vantaggi:
        o       Ottimizzazione per l'italiano: Essendo specificamente addestrato o ottimizzato per l'italiano, offre prestazioni migliori in questa lingua rispetto a modelli general-purpose.
        o       Efficienza: Con soli 2 miliardi di parametri e il formato GGUF, è leggero e può essere eseguito su hardware con risorse limitate, come CPU.
        o       Accessibilità: Il formato GGUF lo rende facilmente utilizzabile da un'ampia gamma di utenti.
    •   Svantaggi:
        o       Capacità limitate: La dimensione ridotta del modello limita la sua capacità di ragionamento complesso e la generazione di testo dettagliato.
        o       Precisione inferiore: Rispetto a modelli più grandi, potrebbe avere una precisione inferiore in compiti complessi.
2. phi4 10b
    •   Vantaggi:
        o       Eccellente ragionamento e codice: Phi-4 è noto per le sue forti capacità di ragionamento e generazione di codice.
        o       Buone prestazioni generali: Con 10 miliardi di parametri, offre un buon equilibrio tra prestazioni e dimensioni.
    •   Svantaggi:
        o       Richiede più risorse: Rispetto a modelli più piccoli, richiede più risorse di calcolo, come GPU.
        o       Meno ottimizzato per l'italiano: Non essendo specificamente ottimizzato per l'italiano, potrebbe avere prestazioni inferiori in questa lingua.
3. gemma-3 12b
    •   Vantaggi:
        o       Prestazioni elevate: Con 12 miliardi di parametri, offre prestazioni elevate in un'ampia gamma di compiti.
        o       Capacità migliorate: Rispetto al Gemma-2-2b, ha una maggiore capacità di ragionamento e generazione di testo.
        o       General Purpose: è un modello che si adatta a svariati utilizzi.
    •   Svantaggi:
        o       Richiede più risorse: Richiede più risorse di calcolo rispetto a modelli più piccoli.
        o       Ottimizzazione per l'italiano parziale: anche se è un modello di Google, come il gemma-2-2b-it, non è specificatamente ottimizzato per la lingua italiana come il modello più piccolo.
4. Qwen1.5-4B-Chat-GGUF
    •   Vantaggi:
        o       Ottimizzato per la conversazione: La versione "Chat" è specificamente progettata per interazioni di dialogo.
        o       Efficienza: Con 4 miliardi di parametri e il formato GGUF, è efficiente e può essere eseguito su CPU.
        o       Accessibilità: Il formato GGUF lo rende facilmente utilizzabile.
    •   Svantaggi:
        o       Capacità limitate: La dimensione ridotta del modello limita la sua capacità di ragionamento complesso.
        o       Meno ottimizzato per l'italiano: Non essendo specificamente ottimizzato per l'italiano, potrebbe avere prestazioni inferiori in questa lingua.

Altro PHP per gestione delle chat

try {
// eliminazione di tutte le chat esistenti dell'utente
$deleteQuery = "DELETE FROM chats WHERE user_id = ?";
$stmt = $con->prepare($deleteQuery);
$stmt->bind_param("i", $userId);
$stmt->execute();

// inserimento di tutte le chat
$insertQuery = "INSERT INTO chats (user_id, title, chat_data) VALUES (?, ?, ?)";
$stmt = $con->prepare($insertQuery);

$con->begin_transaction(); // Inizia una transazione

foreach ($chats as $chat) {
    $title = $chat['name'] ?? 'Chat senza titolo';
    $chatData = json_encode($chat);

    $stmt->bind_param("iss", $userId, $title, $chatData);
    if (!$stmt->execute()) {
        throw new Exception("Errore durante il salvataggio della chat: " . $stmt->error);
    }
}
//l'eliminazione e l'inserimento  si devono fare per essere sicuri che tutte le chat siano aggiornate
$con->commit(); // Conferma la transazione
return ['success' => true, 'message' => 'Tutte le chat salvate con successo'];

} catch (Exception $e) {
$con->rollback(); // Annulla la transazione in caso di errore
return ['success' => false, 'message' => 'Errore: ' . $e->getMessage()];
}


// Funzione per caricare tutte le chat dell'utente
function loadChatsFromDatabase($userId) {
global $con;
//seleziona tutte le chat dell'utente in base all'id e le mette anche in ordine in base alla data di aggiornamento
$query = "SELECT chat_data FROM chats WHERE user_id = ? ORDER BY updated_at DESC";
$stmt = $con->prepare($query);
$stmt->bind_param("i", $userId);
$stmt->execute();
$result = $stmt->get_result();

$chats = [];

if ($result->num_rows > 0) {
while ($row = $result->fetch_assoc()) {
    $chats[] = json_decode($row['chat_data'], true);
}
return ['success' => true, 'chats' => $chats];
} else {
return ['success' => true, 'chats' => [], 'message' => 'Nessuna chat trovata'];
}
}

Sezione PHP che gestisce il DataBase

CREATE TABLE `users` (
`id` int(11) NOT NULL,
`user_id` varchar(20) NOT NULL,
`email` varchar(100) NOT NULL,
`password` varchar(255) NOT NULL,
`date_created` timestamp NOT NULL DEFAULT current_timestamp(),
`admin` tinyint(1) NOT NULL,
`github_id` varchar(100) DEFAULT NULL,
`username` varchar(100) DEFAULT NULL,
`name` varchar(255) DEFAULT NULL
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_general_ci;

CREATE TABLE `chats` (
`chat_id` int(11) NOT NULL,
`user_id` int(11) NOT NULL,
`title` varchar(255) NOT NULL,
`created_at` timestamp NOT NULL DEFAULT current_timestamp(),
`updated_at` timestamp NOT NULL DEFAULT current_timestamp() ON UPDATE current_timestamp(),
`chat_data` longtext NOT NULL
    FOREIGN KEY (`user_id`) REFERENCES `users` (`userid`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_general_ci;
